## 📈 超短趋势自动化交易系统

该系统核心逻辑在于捕捉 **MA5/10/20 多头排列** 的强势动能标的，通过 **2日时间止损** 强制资金周转，结合 **14:40 尾盘决策** 机制，实现高效的超短线套利。

### 2. 四大核心子系统设计

#### 🔍 第一阶段：盘前狙击（扫描与初选）
* **时间**：工作日 08:10
* **核心逻辑**：利用 `uptrend-scanner` 脚本，从全 A 股中筛选前 10 只满足多头排列的股票。
* **技术指标**：$MA_5 > MA_{10} > MA_{20}$ 且成交量配合。
* **产出**：`picks_YYYYMMDD.json`（今日的备选“子弹库”）。

#### ⚡ 第二阶段：日内滚动（监控与执行）
* **时间**：09:40, 10:30, 13:30 (监控)；14:20 (决策执行)
* **策略模型**：
    * **入场控制**：14:20 通过 `investment-decision` 专家合议，只买入“BUY” 或 “ACCUMULATE” 评级且资金流向为正的标的。
    * **仓位管理**：单票 10% 限制，总仓位 75% 警戒线。
    * **退出机制**：
        * **硬性止损**：股价 < MA20 或浮亏 > 5%。
        * **时间止损**：持仓满 2 个交易日且利润在 -1%~1% 之间（弱势震荡），强制卖出。
        * **分步止盈**：达 3%-5% 利润减仓 50%，RSI > 80 动态止盈。

#### 📝 第三阶段：归纳复盘（分析与推送）
* **时间**：工作日 17:00
* **功能**：生成可视化 MD 报告。
* **核心指标**：今日盈亏比、行业集中度、交易执行回溯。
* **推送渠道**：同步至飞书群组，供人工审核与二次判断。

#### 💾 第四阶段：基础设施（数据闭环）
* **时间**：工作日 19:00
* **目的**：确保明早 08:10 运行扫描时，本地 K 线数据库包含今日最新的收盘价、均线数据。
* **降级机制**：Tushare（主） AKShare（备）。

### 3. 系统部署规范（工作区结构）

为确保系统运行不发生 IO 冲突，请保持以下工作目录结构：

```text
/root/.nanobot/workspace/
├── uptrend_picks/      # 存放每日选股 JSON
├── daily_review/       # 存放复盘 MD 文件
├── kline_data/         # 存放 A 股 K 线增量数据
├── trading_log/        # 存放所有阶段的运行日志
└── config/             # 存放 Tushare Token 和雪球配置
```

### 4. 系统运行状态监控建议

作为一个完整的系统，建议您关注以下三个“风险点”：
1.  **数据连续性**：如果 19:00 的 K 线更新失败，明早的 `uptrend-scanner` 结果可能不准确。
2.  **滑点风险**：14:40 介入时，由于是超短标的，建议使用 `snowball-trading` 的限价单或分批拆单功能。
