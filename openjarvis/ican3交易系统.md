# 🤖 自动化交易系统

## 一、 系统运行时间表 (Daily Schedule)

| 时间 | 执行模块 | 任务名称 | 触发频次 | 核心目标 |
| :--- | :--- | :--- | :--- | :--- |
| **08:00** | 盘前情报 | 全球市场简报 | 每天 | 收集关键人物动态及美股数据，定调当日交易基调 |
| **09:45-14:45** | 盘中交易 | 本地QMT做T策略执行 | 每天 (共7个波段) | 监控行情，执行正T/倒T策略，自动调仓 |
| **17:00** | 盘后总结 | 本地QMT持仓复盘 | 工作日 | 分析盈亏、总结操作、评估次日预期并推送飞书 |
| **18:30** | 数据沉淀 | 自动更新A股K线 | 工作日 | 获取今日A股收盘数据，完善系统底层数据库 |


## 二、 核心子系统详解

### 1. 盘前情报中心 (情报获取与大势研判)
* **执行时间**：每日 08:00
* **数据源**：
    * **关键人物动态**：Sam Altman, Greg Brockman, Elon Musk, Donald Trump, Sundar Pichai, Tim Cook, Jensen Huang, Jerome Powell, Mark Zuckerberg (每人最新2篇新闻)。
    * **美股核心标的**：AAPL, GOOGL, AMZN, NVDA, META, TSLA, COHR, MU, WDC, MSFT (实时行情)。
* **输出产物**：`global-market-brief-YYYYMMDD.md`
* **内容要求**：包含AI/大模型、半导体/芯片、云计算/SaaS、消费电子、新能源/电动车、存储/HBM、社交/广告、宏观/利率等板块的影响评估（5级评分），并生成综合交易建议。

### 2. 量化执行引擎 (本地QMT做T)
* **执行时间**：每日 9:45, 10:15, 10:45, 11:15, 13:45, 14:15, 14:45
* **执行逻辑**：
    1.  **基调读取**：加载今日《全球市场简报》确立市场风向。
    2.  **量化扫描**：调用底层 Python 脚本 `t_trading_scanner.py` 进行技术面扫描。
    3.  **自动交易**：根据系统生成的 LONG/SHORT 信号，调用 `local-qmt-trading skill` 执行调仓操作。
* **严格纪律与风控**：
    * **时间窗口管理**：
        * `09:30-10:00`：观察（避免开盘波动）
        * `10:00-10:30`：**第一黄金窗口**（正T/倒T）
        * `10:30-13:30`：观察（少操作）
        * `13:30-14:00`：**第二黄金窗口**（正T/倒T）
        * `14:00-14:30`：减仓（逐步降低仓位）
        * `14:30之后`：禁止新开（只平仓）
    * **交易口诀（量化指标化）**：
        * **正T**：J值 < 25 且缩量 -> 买入；触及下轨 -> 买入；盈利 2% -> 卖出；亏损 1.5% -> 坚决止损。
        * **倒T**：J值 > 80 且放量 -> 卖出；下跌 2% -> 买回不追高；突破 2% -> 放弃，持有现金。
        * **全局熔断**：大盘跌超 2% -> 停止所有开仓；个股跌停 -> 剔除交易池；连续亏损3次 -> 强制冷静暂停；尾盘半小时绝对不新开仓。

### 3. 数据与复盘中心 (迭代与进化)
* **17:00 账户复盘监控**：
    * 核算本地QMT持仓与资金变动。
    * 分析持仓盈亏及行业分布健康度。
    * 深度回顾当日交易操作，结合大势预判明日风险。
    * **消息路由**：生成 `daily_review_YYYYMMDD.md` 并自动推送至 **飞书(Feishu)** 进行移动端提醒。
* **18:30 行情数据清洗与入库**：
    * 调用 `fetch_kline.py` 增量拉取A股今日收盘数据。
    * **高可用切换**：优先调用 Tushare API，若 Token 失效或限流，自动降级切换至免费 AKShare。
    * 进行完整性和格式(CSV)验证，确保量化回测底座数据的干净可靠。


## 三、 系统文件目录结构规范 (Workspace Structure)

系统运行依赖以下标准化的目录结构来管理产出：

```text
/root/.nanobot/workspace/
│
├── reports/                      # 盘前情报存储区
│   └── global-market-brief-YYYYMMDD.md
│
├── t_trading_report/             # 盘中做T策略研判存储区
│   └── t_trading_report-YYYYMMDD-hhmm.md
│
├── trading_log/                  # 系统执行与交易流水日志
│   ├── t_trading_YYYYMMDD-hhmm.md       # 做T操作流水
│   └── kline_update_YYYYMMDD.md         # 行情入库日志
│
├── daily_review/                 # 盘后深度复盘报告
│   └── daily_review_YYYYMMDD.md
│
└── kline_data/                   # A股基础K线数据库
    └── *.csv                     # 自动入库的清洗后数据
```
