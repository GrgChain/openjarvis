# 🚀 全自动化量化交易系统 

## 一、 系统核心价值观与交易铁律 (The Core Rules)

*“纪律是交易生存的唯一保障。”*

系统在任何自动执行环节，都**绝对服从**以下四大资金与风控约束：

### 1. 仓位硬性红线 (Position Management)

  * **永不满仓**：系统设定总仓位目标上限为 **75%**，始终保留至少 **25%** 的现金作为战略预备队。
  * **单票熔断**：单只股票持仓**绝对禁止超过 10%**，强制执行风险分散。
  * **禁买隔离带**：当总仓位一旦触碰 **80%**，系统将触发“只卖不买”模式，拒绝任何新开仓指令。

### 2. 流动性防线 (Liquidity Guard)

  * **极寒警报**：当账户现金比例低于 **20%** 时，系统冻结所有买入权限。
  * **强制抽血**：触发警报后，系统按顺序无条件裁撤：**弱势股 -> 无主力资金关注股 -> 已破位个股**，直至现金水位恢复至 25% 以上。

### 3. 无情止盈止损 (Exit Strategy)

  * **静态止损**：一旦跌破关键技术位或触及 **-5% ~ -8%** 的亏损线，系统将无条件斩仓，杜绝任何“扛单”行为。
  * **动态止盈**：当个股达到预期目标或上攻动能出现衰竭特征时，系统执行分批止盈（首批撤退 50%），并启动移动止盈策略保护剩余利润。

### 4. 铁血周期管理 (Time-Cycle Management)

  * **持仓保质期**：原则上单票持仓**不超过 5 个交易日**。
  * **时间止损**：满 5 个交易日仍未触发止盈/止损的标的，除非处于“均线多头排列的极强势主升浪”，否则一律在尾盘（14:30任务）强制平仓，以保证资金的极致周转率。

## 二、 自动化运行矩阵 (Daily Automated Pipeline)

系统在工作日按时间轴执行以下 5 大核心模块，形成“数据获取 -> 选股 -> 决策 -> 交易 -> 复盘”的完美闭环。

| 执行时间 | 任务模块 | 核心动作与逻辑 | 关联系统组件/脚本 |
| :--- | :--- | :--- | :--- |
| **09:50**  **10:30**  **13:50** | **盘中动态调仓** | 1. 监控仓位与 20% 现金红线  2. 扫描 -5%\~-8% 止损位与衰竭止盈点  3. 基于昨日 20:00 决策库执行新仓买入  4. 执行 5 日持仓周期审查 | `snowball-trading`  日志: `dynamic_trade_YYYYMMDD.md` |
| **14:30** | **尾盘风控清算** | 1. 执行上述盘中调仓逻辑  2. **末路检查点**：无条件强制清理满 5 日且非强势的陈旧仓位，为明日释放流动性 | 同上 |
| **17:00** | **雪球每日复盘** | 1. 统计持仓盈亏与行业分布  2. 回顾当日操作得失  3. 推演明日市场预期与潜在风险  4. 生成报告并推送至**飞书** | 日志: `daily_review_YYYYMMDD.md` |
| **18:00** | **K线数据更新** | 1. 增量获取今日 A 股收盘数据  2. 优先 Tushare，无 Token 降级 AKShare  3. 校验数据完整性与 CSV 格式 | `fetch-kline`  数据: `workspace/kline_data` |
| **19:00** | **量化多维选股** | 1. 扫描全市场 A 股  2. 运用 BBI, KDJ, MA60, 成交量等多策略组合筛选  3. 生成策略候选池 JSON | `stock-selection`  数据: `stock_picks/picks_今日.json` |
| **20:00** | **共振深度决策** | 1. **策略共振**：筛选命中 ≥2 个策略的个股  2. **板块赋能**：叠加当日强势板块(得分≥60)  3. 行业强度补齐至 20 只  4. 叠加 `risk-assessment` 专家一票否决制  5. 生成明日入场点与止损/止盈位 | `investment-decision`  决策: `investment_decision_YYYYMMDD.json` |

## 三、 数据流转与目录架构 (Data Architecture)

系统高度依赖本地工作区的标准目录结构进行信息传递：

  * 📁 `/root/.nanobot/workspace/`
      * 📂 `kline_data/` —— **数据底座**：存放 18:00 更新的每日行情 CSV 数据。
      * 📂 `stock_picks/` —— **粗筛标的**：存放 19:00 量化初选生成的 JSON 列表。
      * 📂 `sector/` —— **板块热度**：存放板块强度得分（\>=60分）的数据记录。
      * 📂 `decisions/` —— **作战指令**：存放 20:00 深度分析后生成的 JSON 格式买卖计划（直接供次日调仓任务读取）。
      * 📂 `daily_review/` —— **复盘归档**：存放 17:00 生成的每日图文复盘 Markdown 报告。
      * 📂 `trading_log/` —— **系统日志**：
          * `kline_update_*.md` (数据更新日志)
          * `stock_selection_*.md` (选股策略日志)
          * `investment_analysis_*.md` (决策分析详志)
          * `dynamic_trade_*.md` (实盘交易动作日志)

## 四、 策略决策引擎逻辑流 (Decision Logic Flow)

20:00 的深度决策是系统的“大脑”，其优先级逻辑严格遵循：
**多策略共振 (高胜率) > 单策略+强势行业 (高赔率) > 行业综合强度补位 (保容错)**

1.  **海选阶段**：通过技术面（BBI/KDJ/MA60等）初筛。
2.  **共振筛选**：寻找技术指标重合度高的标的。
3.  **风控阻断**：每一只入围股票必须经过风控模型的“一票否决”审查（排雷基本面、消息面恶化）。
4.  **生成弹药库**：将存活下来的 20 只股票标的，附带明确的【买入区间、止损价、止盈预期】，写入 `investment_decision_YYYYMMDD.json` 供次日使用。
